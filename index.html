SOLID

5 principios, 5 letras, 1 objetivo.

S — Single Responsibility

Una clase, una razon para cambiar.

Cada modulo hace una sola cosa. Si cambian dos cosas juntas, estan mal separadas.

// MALO: hace todo
class UserService {
  login() { }
  saveToDb() { }
  sendEmail() { }
}

// BUENO: responsabilidad unica
class AuthService {
  login() { }
}
class UserRepository {
  save() { }
}
class EmailService {
  send() { }
}

O — Open/Closed

Abierto para extension, cerrado para modificacion.

Agrega comportamiento sin tocar el codigo existente.

// MALO: modifcar para agregar
class Discount {
  apply(type, price) {
    if (type == 'vip') return price * 0.8;
    if (type == 'normal') return price * 0.9;
  }
}

// BUENO: extender sin modificar
abstract class Discount {
  apply(price);
}
class VipDiscount implements Discount {
  apply(price) => price * 0.8;
}

L — Liskov Substitution

Subtipos deben ser sustituibles por sus tipos base.

Si B es subtipo de A, debe funcionar sin romper nada.

// MALO: rompe el contrato
class Bird {
  fly() { }
}
class Penguin extends Bird {
  fly() { throw Error("No puedo"); }
}

// BUENO: interfaces correctas
abstract class Bird {
  move();
}
class FlyingBird extends Bird {
  move() => this.fly();
}
class SwimmingBird extends Bird {
  move() => this.swim();
}

I — Interface Segregation

Ningun cliente debe depender de metodos que no usa.

Interfaces pequenas y especificas.

// MALO: interfaz gigante
interface Worker {
  work();
  eat();
  sleep();
}

// BUENO: interfaces segregadas
interface Workable { work(); }
interface Eatable { eat(); }
class Robot implements Workable {
  work() { }
}
class Human implements Workable, Eatable {
  work() { }
  eat() { }
}

D — Dependency Inversion

Depende de abstracciones, no de concreciones.

Los modulos de alto nivel no deben depender de los de bajo nivel.

// MALO: dependencia concreta
class UserRepository {
  final db = MySqlDatabase(); // atado a MySQL
}

// BUENO: depende de abstraccion
abstract class Database {
  save(String table, Map data);
}
class UserRepository {
  final Database db; // puede ser MySQL, Postgres, Mock
  UserRepository(this.db);
}

// RED: test que falla
test('should return default greeting', () async {
  final repo = GreetingRepositoryImpl();
  final greeting = await repo.getGreeting();
  expect(greeting.message, 'Hola Mundo');

// GREEN: implementar minimo

Fundamentos del diseno

  • Claridad: Interfaz intuitiva sin ambigüedades

Psicologia del color en UX

Verde: Exito, crecimiento, accion positiva

Azul: Confianza, seguridad, profesionalismo

Morado: Creatividad, innovacion, premium

Oscuro: Sophistication, modernidad, foco

60-30-10 Rule

60% Superficie (background)
30% Secundario (cards, borders)
10% Acento (CTAs, highlights)

Dark Mode

No es solo invertir colores. Usar grises oscuros (#1A1A2E) en vez de negro puro para reducir fatiga visual.

Tipografia

Jerarquia tipografica

Heading 1 (32px Bold)

Heading 2 (24px Semibold)

Heading 3 (20px Medium)

Body Text (16px Regular) - El tamano base para legibilidad optima en movil.

Caption (14px) - Para labels, metadata y texto secundario.

Font Pairing

// Inter + Fira Code (Nuestro stack)
Headings:    Inter (700, 600)
Body:        Inter (400, 500)
Code:        Fira Code (400)

// Google Fonts en Flutter
GoogleFonts.inter(
  fontSize: 16,
  fontWeight: FontWeight.w400,
  height: 1.5, // line-height
)

Line Height

1.4-1.6 para body text. Mucho espacio = mas legibilidad. Testing con usuarios reales.

Glassmorphism en Flutter

// Efecto Glassmorphism
ClipRRect(
  borderRadius: BorderRadius.circular(16),
  child: BackdropFilter(
    filter: ImageFilter.blur(
      sigmaX: 16,
      sigmaY: 16,
    ),
    child: Container(
      decoration: BoxDecoration(
        color: Colors.white.withOpacity(0.05),
        borderRadius: BorderRadius.circular(16),
        border: Border.all(
          color: Colors.white.withOpacity(0.1),
          width: 1,
        ),
      ),
      child: child,
    ),
  ),
)

Ejemplo Visual

Este card usa glassmorphism: fondo semi-transparente, blur, y borde sutil. Crea profundidad sin sombras pesadas.
